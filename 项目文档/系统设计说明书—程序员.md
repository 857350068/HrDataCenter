# <a name="_toc376255122"></a><a name="_toc376255405"></a><a name="_toc27151"></a>**1. Introduction  简介**
## <a name="_toc376255123"></a><a name="_toc376255406"></a><a name="_toc31946"></a>**1.1 Purpose  文档目的**
本文档是HrDataCenter人力资源数据中心系统的系统设计说明书，旨在详细描述系统的架构设计、数据库设计以及各功能模块的详细设计，为开发人员提供实现指导，并确保系统的可维护性和可扩展性。

## <a name="_toc376255124"></a><a name="_toc376255407"></a><a name="_toc31703"></a>**1.2 Scope  本文档适用范围** 
本文档适用于HrDataCenter人力资源数据中心系统的设计阶段，涵盖系统架构设计、数据库设计以及核心功能模块的详细设计。本文档面向系统架构师、开发人员、测试人员以及项目管理人员，为系统开发和维护提供技术参考。

## <a name="_toc376255125"></a><a name="_toc376255408"></a><a name="_toc29414"></a>**1.3 Name 软件名称** 
人力资源数据中心(HrDataCenter)

## <a name="_toc376255411"></a><a name="_toc9028"></a>**1.4 Applications软件应用领域**
本系统主要应用于企业人力资源管理领域，为企业提供员工档案管理、数据分析、预警提醒、报表生成等功能，帮助企业实现人力资源数字化转型。

# <a name="_toc376255127"></a><a name="_toc15096"></a>**2. High Level Design概要设计**
## <a name="_toc376255128"></a><a name="_toc32036"></a>**2.1系统功能设计**
HrDataCenter人力资源数据中心系统主要划分为以下功能模块：

1. **用户认证模块**：用户登录、注册、权限管理
2. **员工档案管理模块**：员工基本信息录入、查询、修改、删除
3. **数据分析模块**：包含8个数据分析子模块
   - 组织效能分析
   - 人才梯队分析
   - 薪酬福利分析
   - 绩效管理分析
   - 员工流失预警
   - 培训效果评估
   - 人力成本优化
   - 人才发展预测
4. **预警规则模块**：配置和管理各类预警规则
5. **报表模板模块**：自定义报表模板
6. **收藏功能模块**：用户收藏常用分析
7. **系统管理模块**：部门管理、用户管理、角色权限管理

## <a name="_toc376255129"></a><a name="_toc20408"></a>**2.2 系统架构设计**
### <a name="_toc376255130"></a><a name="_toc11751"></a>**2.2.1 三层架构**
本项目采用三层架构进行开发，其优势在于便于分工合作、易于扩展、便于维护，其三层之间的关系如下所示。其中表示层用于与用户进行交互；业务逻辑层负责处理项目业务流程；数据访问层负责操作数据库；Model层中的实体类是三层之间的数据载体，用于在三层之间传递数据。

**三层架构说明：**

1. **表示层（Presentation Layer）**
   - 前端框架：Vue 3.4.0 + Element Plus
   - 负责用户界面展示和交互
   - 通过HTTP/HTTPS协议与后端通信
   - 使用RESTful API进行数据交互

2. **业务逻辑层（Business Logic Layer）**
   - 后端框架：Spring Boot 2.7.18
   - 负责业务逻辑处理
   - Service层实现核心业务功能
   - Controller层处理HTTP请求

3. **数据访问层（Data Access Layer）**
   - 持久化框架：MyBatis Plus 3.5.5
   - 负责数据库操作
   - Mapper层实现数据CRUD
   - 支持MySQL和Hive双数据源

4. **安全认证层**
   - Spring Security + JWT
   - 实现用户认证和授权
   - 支持基于角色的访问控制(RBAC)

# <a name="_toc376255412"></a><a name="_toc14109"></a>**3. Low Level Model Design 模块详细设计**
## <a name="_toc376255413"></a><a name="_toc15185"></a>**3.1 User Model Design 用户模块**
<a name="_toc9959"></a>**(1)Process analysis业务流程分析**

用户登录模块的业务流程如下：

1. 首先在表示层界面中输入用户名和密码
2. 然后在业务逻辑层中进行身份校验
3. 在进行身份校验时，调用数据访问层查询用户信息进行身份比对
4. 验证通过后生成JWT Token返回给前端
5. 前端存储Token并在后续请求中携带
6. 后端通过Spring Security验证Token有效性

<a name="_toc13952"></a>**(2)Flow Diagram流程图**

用户登录流程图：

```
用户输入用户名密码
      ↓
前端发送登录请求
      ↓
Controller接收请求
      ↓
UserService处理登录逻辑
      ↓
UserMapper查询用户信息
      ↓
验证密码是否正确
      ↓
生成JWT Token
      ↓
返回登录结果给前端
```

## <a name="_toc376255418"></a>**3.3 核心业务模块设计**

### <a name="_toc376255419"></a>**3.3.1 组织效能分析模块**

**功能描述**：分析企业组织架构的效能，包括部门人员分布、岗位配置合理性等。

**核心类**：
- OrganizationAnalysisService：组织效能分析服务类
- OrganizationAnalysisController：组织效能分析控制器类

**主要方法**：
- analyzeDeptDistribution()：分析部门人员分布
- analyzePositionStructure()：分析岗位结构
- generateReport()：生成分析报告

### <a name="_toc376255420"></a>**3.3.2 人才梯队分析模块**

**功能描述**：分析企业人才梯队建设情况，识别关键岗位继任者。

**核心类**：
- TalentAnalysisService：人才梯队分析服务类
- TalentAnalysisController：人才梯队分析控制器类

**主要方法**：
- analyzeTalentPipeline()：分析人才梯队
- identifyKeyTalent()：识别关键人才
- assessSuccessionReadiness()：评估继任准备度

### <a name="_toc376255421"></a>**3.3.3 员工流失预警模块**

**功能描述**：基于历史数据和规则，预警可能离职的员工。

**核心类**：
- TurnoverWarningService：流失预警服务类
- TurnoverWarningController：流失预警控制器类

**主要方法**：
- calculateTurnoverRisk()：计算流失风险
- generateWarningList()：生成预警名单
- sendAlert()：发送预警通知

## <a name="_toc376255426"></a>**3.5 接口设计规范**

### <a name="_toc376255427"></a>**3.5.1 RESTful API设计**

系统采用RESTful API设计风格，遵循以下规范：

- **GET**：查询资源
- **POST**：创建资源
- **PUT**：更新资源
- **DELETE**：删除资源

**接口示例**：

```
POST /api/auth/login          # 用户登录
GET  /api/users               # 获取用户列表
GET  /api/users/{id}          # 获取用户详情
POST /api/users               # 创建用户
PUT  /api/users/{id}          # 更新用户
DELETE /api/users/{id}        # 删除用户
GET  /api/analysis/org        # 组织效能分析
GET  /api/analysis/talent     # 人才梯队分析
POST /api/warnings            # 创建预警规则
GET  /api/reports             # 获取报表列表
```

### <a name="_toc376255428"></a>**3.5.2 统一响应格式**

所有接口统一使用以下响应格式：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    // 业务数据
  },
  "timestamp": "2024-12-01T10:00:00"
}
```

**状态码说明**：
- 200：操作成功
- 400：请求参数错误
- 401：未授权
- 403：禁止访问
- 404：资源不存在
- 500：服务器内部错误
